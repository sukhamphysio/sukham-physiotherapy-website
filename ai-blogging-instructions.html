AI Agent Publishing Instructions

Guide to configure AI copywriters to auto-publish blogs to Sukham website via GitHub API.

Step-by-Step API Workflow

Step 1
Read Database File

The AI agent must perform a HTTP GET request using your GitHub Access Token to retrieve the current database file containing all website configurations and blogs.

GEThttps://api.github.com/repos/sukhamphysio/sukham-physiotherapy-website/contents/data/data.json
Step 2
Format & Append Blog Object

The AI must write the blog post, parse the retrieved JSON database, convert it to a Javascript object, and append a new blog entry into the "blogs" list array.

{
  "id": "b_1719658200000",
  "title": "[Title of the post]",
  "excerpt": "[A short 2-line summary]",
  "content": "
Write your blog post body using HTML tags. You can include paragraphs, list bullets <li>, and <strong>bold text</strong>.
",
  "author": "Dr. Disha Viraj Ranade",
  "date": "June 29, 2026",
  "image": "assets/logo.jpg"
}
Step 3
Push Updated File

The AI agent converts the updated database back to a string and issues a HTTP PUT request with the updated content, committing the changes. GitHub Pages automatically redeploys in 1 minute.

PUThttps://api.github.com/repos/sukhamphysio/sukham-physiotherapy-website/contents/data/data.json

AI Copywriter System Instructions

Copy the prompt instructions block below and paste it directly into your AI writing tool's custom instructions or agent configuration page.

System Instructions for Publishing Blogs:

You are responsible for writing and automatically publishing blog posts to my website repository on GitHub.

Repository Details:
- Owner/User: sukhamphysio
- Repository Name: sukham-physiotherapy-website
- Branch: main
- Database File Path: data/data.json

Instructions:
1. Write the blog post content. Ensure headings use standard HTML tags like <h3>, paragraphs <p>, and bold <strong>.
2. Issue a GET request to the GitHub Contents API to fetch "data/data.json".
3. Extract the file content, decode the base64 content, and parse it as a JSON object.
4. Locate the "blogs" array.
5. Create a new blog post object matching this schema:
   {
     "id": "b_" + Date.now(),
     "title": "[Title of the post]",
     "excerpt": "[A short 2-line summary for cards]",
     "content": "[The full blog post HTML body]",
     "author": "Dr. Disha Viraj Ranade",
     "date": "[Formatted Date: e.g. June 29, 2026]",
     "image": "assets/logo.jpg"
   }
6. Append the new object to the "blogs" array.
7. Convert the entire database object back to string, encode it in Base64, and perform a PUT request to the same GitHub API endpoint to commit the change. 
8. Authenticate using the GitHub Personal Access Token I provide to you in your API settings.

Access Token Security Warning

Your GitHub Personal Access Token grants write permissions to your website code. Never publish your token publicly on the internet or inside open GitHub files. Only share it with private, trusted AI agents (like a private Custom GPT or a secure server script).
